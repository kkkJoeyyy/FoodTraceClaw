# MVP Food Chat

## Why

用户在抖音/小红书看到美食推荐时分散保存，事后难检索。需要一个工具按**店铺 → 菜品**两层结构记录美食，按地点推荐 Top 5 店铺。

## What

一个 Web 聊天助手：粘贴分享文案/截图后 LLM 自动提取店铺名+菜品+地点入库；询问"XX 有什么好吃的"时按地点推送 Top 5 店铺（每店含招牌菜），支持翻页。

## Scope

- 后端 FastAPI + sqlite3
- 前端 React Chat 界面
- LLM 提取店铺 + 菜品 (OpenAI/Ollama 双通道)
- VL 图片提取
- 地理编码 + 距离排序
- 店铺去重：同名+近位置 → 复用已有店铺，追加新菜品
- Top 5 店铺推荐 + 翻页
- Docker Compose 一键部署

## Non-goals

- IP 定位（MVP 不做）
- 微信 Bot 接入（后续迭代）
- 抖音/小红书链接自动抓取
- 用户认证/多用户
- 评分/评论系统
